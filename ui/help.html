Washes Nearby

Pushes CRM washes to the public map via /api/v1/integration/*.

Setup

  1. Get an Owner API token in the Washes Nearby owner cabinet.
  2. Set the site / API URL, send interval, news limit, and finance timezone.
  3. In the Washes block, set coordinates, city and type for each CRM wash.
  4. Save and start the module.

Finance (cash register)

Every telemetry cycle (≥ 60 s) the module reads /api/crm/finance-stats and adds a finance block: cash / external / discount for today, before and after collection — wash-level and per post.

  • before_collection / after_collection — live CRM counters (cash, cashless→external, discountOps→discount).
  • today — day growth of cumulative after_collection in the configured timezone.
  • 7-day and 30-day views on the site are summed from daily today — the module does not send those aggregates.

Prices

For each work mode code, the module sends the maximum price across posts. Names come from /api/crm/work-modes.

Coordinates

Set latitude/longitude per wash in the list. Shared default coordinates are no longer applied automatically — that used to make two washes share one map point.

The module only PATCHes coordinates when they change for that wash. Re-sending the same lat/lng should not force moderation again.

Occupancy

Telemetry sends post status every cycle (≥ 60 s). Prices update via PATCH only when content changes.

News & promotions

From CRM Publications: news/general → news, promotion → promotions.
